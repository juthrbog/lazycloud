# S3

Browse, preview, download, copy, move, and manage S3 buckets and objects.

## Views

### Bucket List

Lists all S3 buckets accessible to the current profile/credentials.

| Column | Description |
|--------|-------------|
| Bucket | Bucket name |
| Created | Creation date |

### Object Browser

Browse objects within a bucket with folder-style navigation using S3 prefixes. Objects load progressively (1000 per page) so the first results appear immediately.

| Column | Description |
|--------|-------------|
| Type | Folder or file icon |
| Name | Object key (relative to current prefix) |
| Size | Human-readable file size |
| Modified | Last modified date |
| Class | Storage class (STANDARD, GLACIER, etc.) |

### Version List

View all versions of a specific object (requires versioning enabled on the bucket).

| Column | Description |
|--------|-------------|
| Version ID | S3 version identifier |
| Size | Version size |
| Modified | Version timestamp |
| Latest | Whether this is the current version |
| Delete Marker | Whether this is a delete marker |

## Keybindings

### Bucket List

| Key | Action |
|-----|--------|
| `enter` | Browse bucket contents |
| `d` | Show bucket properties (versioning, encryption, public access, region) |
| `n` | Create new bucket |
| `x` | Delete bucket (empties all objects first, requires confirmation) |
| `/` | Filter buckets |
| `r` | Refresh |

### Object Browser

| Key | Action |
|-----|--------|
| `enter` | Drill into folder / preview file content |
| `d` | Show object metadata (HeadObject) as JSON |
| `w` | Download file to `~/Downloads/` |
| `c` | Copy object (prompts for destination path) |
| `m` | Move/rename object (copy + delete source) |
| `v` | View object versions |
| `u` | Generate presigned URL (1hr) and copy to clipboard |
| `y` | Copy `s3://bucket/key` path to clipboard |
| `space` | Toggle multi-select on current row |
| `x` | Delete selected objects (or current if none selected, requires confirmation) |
| `/` | Filter objects |
| `r` | Refresh |

### Version List

| Key | Action |
|-----|--------|
| `enter` | View version details as JSON |
| `d` | View version details as JSON |
| `r` | Refresh |

## Content Preview

Pressing `enter` on a file performs a HEAD check to determine if the file is previewable:

- **Text files** (by ContentType or extension): content is fetched (up to 1MB) and displayed with syntax highlighting in the content viewer
- **Binary or large files**: object metadata is shown as a fallback

Supported text extensions: `.txt`, `.json`, `.yaml`, `.yml`, `.xml`, `.html`, `.css`, `.js`, `.ts`, `.go`, `.py`, `.sh`, `.md`, `.csv`, `.log`, `.conf`, `.toml`, `.tf`, `.hcl`, `.sql`, and more.

## Multi-Select

Press `space` to select/deselect objects. Selected items show a `✓` marker. The status bar shows the selection count. Press `x` to delete all selected items (requires typing "confirm").

Selections persist across filter changes — you can select items, filter, select more, then delete all selected.

## Bucket Operations

- **Create** (`n`): enter a bucket name, creates in the current region
- **Properties** (`d`): shows versioning status, encryption, public access block, and region
- **Delete** (`x`): empties all objects then deletes the bucket (requires typing "confirm")

## Cross-Region Support

Buckets are automatically accessed in their correct region. LazyCloud resolves each bucket's region via `GetBucketLocation` to avoid 301 redirect errors when the bucket is in a different region than the configured client.

## LocalStack

Works with LocalStack. Seed test buckets with:

```bash
task localstack:seed
```

This creates `test-bucket-1`, `test-bucket-2`, and `logs-bucket` with sample files across multiple folders and file types.
