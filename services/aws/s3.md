# S3

Browse, preview, and manage S3 buckets and objects.

## Views

### Bucket List

Lists all S3 buckets accessible to the current profile/credentials.

| Column | Description |
|--------|-------------|
| Bucket | Bucket name |
| Created | Creation date |

### Object Browser

Browse objects within a bucket with folder-style navigation using S3 prefixes. Objects load progressively (1000 per page) so the first results appear immediately.

| Column | Description |
|--------|-------------|
| Type | Folder or file icon |
| Name | Object key (relative to current prefix) |
| Size | Human-readable file size |
| Modified | Last modified date |
| Class | Storage class (STANDARD, GLACIER, etc.) |

## Keybindings

### Bucket List

| Key | Action |
|-----|--------|
| `enter` | Browse bucket contents |
| `d` | Show bucket metadata as JSON |
| `/` | Filter buckets |
| `r` | Refresh |

### Object Browser

| Key | Action |
|-----|--------|
| `enter` | Drill into folder / preview file content |
| `d` | Show object metadata (HeadObject) as JSON |
| `u` | Generate presigned URL (1hr) and copy to clipboard |
| `y` | Copy `s3://bucket/key` path to clipboard |
| `/` | Filter objects |
| `r` | Refresh |

## Content Preview

Pressing `enter` on a file performs a HEAD check to determine if the file is previewable:

- **Text files** (by ContentType or extension): content is fetched (up to 1MB) and displayed with syntax highlighting in the content viewer
- **Binary or large files**: object metadata is shown as a fallback

Supported text extensions: `.txt`, `.json`, `.yaml`, `.yml`, `.xml`, `.html`, `.css`, `.js`, `.ts`, `.go`, `.py`, `.sh`, `.md`, `.csv`, `.log`, `.conf`, `.toml`, `.tf`, `.hcl`, `.sql`, and more.

## Cross-Region Support

Buckets are automatically accessed in their correct region. LazyCloud resolves each bucket's region via `GetBucketLocation` to avoid 301 redirect errors when the bucket is in a different region than the configured client.

## LocalStack

Works with LocalStack. Seed test buckets with:

```bash
task localstack:seed
```

This creates `test-bucket-1`, `test-bucket-2`, and `logs-bucket` with sample data.
